💡 研究動機與背景

南投縣（如集集、中寮、草屯等鄉鎮）是台灣特有種「石虎」最核心的棲地。然而，山區道路與棲地高度重疊，導致夜間「路殺」（Roadkill）事件頻繁發生。石虎屬於夜行性動物，其身上的斑紋在柏油路上具有極佳的保護色，在缺乏照明的山區道路，駕駛者往往受限於視覺死角，在發現目標時已避讓不及。這不僅造成珍貴物種的消逝，更嚴重威脅行車安全。

透過資料搜集，我了解到現行車輛的主動避障技術多依賴「超音波」或「影像辨識」。然而，超音波在高速行駛下的精準度受限，而影像辨識在極端全黑、強光或保護色干擾下容易產生誤判。為了解決這個痛點，我對「單點光學雷達（ToF 測距模組）」產生了濃厚興趣。光學雷達具備不受環境光干擾、反應速度快且測距精準的優勢。

本研究結合 MicroPython 程式撰寫，試圖在縮小比例的實驗環境中，模擬石虎在夜間出沒的情境。我參考了「送餐機器人」規劃路徑的原理，導入「網格座標路徑規劃」演算法，希望探討系統如何在「及時煞車」與「動態避障」之間找到最佳平衡點。

🏆 科學實證：加入「超音波」對照組之必要性

為避免流於紙上談兵，本研究特別引入市面普及的 HC-SR04P 超音波感測器 作為科學對照組。我們不在所有情境浪費資源，而是「精準打擊」，針對石虎的生物特性（深色毛皮）與棲地特性（狹窄山路），設計了殘酷的雙感測器極限對決。透過數據實證，我們將親自證明為何傳統超音波無法勝任，而光學雷達是預防路殺的唯一正解！

🌟 預期成果與貢獻

本研究預期能證實光學雷達在夜間完全黑暗環境下，測距精度優於傳統紅外線或超音波感測器。透過「路殺預警系統」的雛型開發，結合自動緊急煞車（AEB）與動態繞道邏輯，為未來智慧交通提供一個平衡「生物保育」與「行駛效率」的技術藍圖。

🚀 未來進一步研究方向

考量戶外山區電力匱乏，未來本系統需搭配**獨立太陽能綠電系統（含電池儲能與發電度數 kWh 監控）**。此外，計畫導入 AI 影像辨識鏡頭（如 HuskyLens），透過「雷達測距」與「影像辨識」的感測器融合（Sensor Fusion），實現更高層級的精準守護。

📝 我的實驗計畫與設計

🎯 研究目的

本研究旨在探討單點光學雷達（LiDAR）應用於夜間野生動物防路殺系統的可行性，具體研究目的如下：

  • 驗證環境光源對測距的影響： 測試光學雷達在極端光源下的穩定性。
  • 探討保護色與材質的極限（對照測試）： 對比光學雷達與超音波在面對「深色絨毛（模擬石虎）」時的效能差異。
  • 優化感測模組的最佳視角： 找出最適合精準偵測低矮動物的雷達高度。
  • 建立安全預警與煞車門檻： 利用雷達自動測量小車在不同速度下的極限煞車滑行距離。
  • 多情境動態避障與誤判分析（對照測試）： 實作路徑規劃，並驗證超音波在「狹窄山路」中易受兩側環境干擾而誤判的致命缺陷。

🎯 針對性對照組策略：為何只在實驗二與五-A加入超音波？

在科學研究中，合理的資源分配與針對性測試至關重要。我們並不盲目地在所有實驗中加入超音波對照組，而是基於物理原理進行以下精準打擊：

  • ❌ 不比實驗一（光線）與三（高度）： 超音波利用聲音測距，不受光線影響；且其扇形音波難以像雷射般精準控制掃描高度，對比意義不大。
  • ❌ 不比實驗四（煞車距離）： 煞車滑行距離取決於車體重量與輪胎摩擦力的「物理極限」，與使用何種感測器下達停止指令無關。
  • ⚔️ 主戰場 實驗二（材質）： 光學雷達怕「黑色吸光」，超音波怕「軟毛吸音」。石虎恰好具備「深色且毛茸茸」的特性，這是測試兩種感測器生死極限的最佳舞台！
  • ⚔️ 主戰場 實驗五-A（模擬山路）： 雷射光是「直線」，超音波是「扇形」。在狹窄山路中，超音波極易撞擊兩側山壁或樹木產生假警報（誤判），這正是我們必須用實驗揭露的超音波致命傷！

🧰 研究設備與器材

  • 核心大腦： micro:bit V2.2 微控制器。
  • 實驗載具： 麥昆 Plus V3 智能擴充小車。
  • 測距主角： VL53L0X 單點光學雷達模組（ToF 感測器）。
  • 對照組感測器： HC-SR04P 超音波感測器模組（用於凸顯光學雷達優勢）。
  • 程式語言： MicroPython。
  • 模擬道具： 白/棕紙板、深色絨毛玩偶、捲尺、碼表、網格地墊。

🔍 實驗一：夜盲症測試（光線變因）

分別在正常日光燈、手電筒強光與關燈全黑三種環境下，測試光學雷達於 30~150 cm 的測距精準度。

🔍 實驗二：保護色與材質陷阱（材質變因）＋【雙感測器對決】

  • 步驟： 於筆直軌道上設定 30~150 公分距離。
  • 測試： 對比白色紙板（平滑高反光）、棕色紙板（霧面漫反射）與黑色絨毛玩偶（深色吸光吸音）對感測器的影響。
  • 🔥 觀察重點： 同步紀錄雷達與超音波的測量誤差，驗證面對「深色絨毛」時，何者會率先失效產生盲區。

🔍 實驗三：最佳視角測試（高度變因）

將雷達安裝於 5cm、10cm、15cm 高度，測試最能穩定偵測低矮型野生動物的硬體配置。

🔍 實驗四：生死一瞬間（自動測量煞車距離）

設定小車以 20~60 cm/s 前進。撰寫自動化測試程式：利用雷達在 100cm 處觸發煞車，待車輛停止後，再次透過雷達測量最終距離。兩次距離相減即為「極限物理滑行距離」，徹底消除人為量測誤差。

🔍 實驗五：生命轉彎處（多情境動態路徑規劃）＋【山路抗干擾對決】

  • 情境 A：受限車道（模擬山路） 測試小車在白線內閃避石虎的能力。**對照測試：**換上超音波，驗證其「扇形音波」是否會掃到兩側牆壁而導致假警報煞車。
  • 情境 B：開放空地（24 宮格地圖） 測試九宮格視野的網格最短避障演算法。

2. 小車端（發射端）最終優化版程式碼

這是我們使用 MicroPython 撰寫的核心大腦！裡面包含了 LiDAR 讀取、毫米換算公分、計算平均值，以及為了解決通訊碰撞而加入的 200ms 無線電排隊廣播機制。

def on_button_pressed_a():
    global total_distance, average, avg_msg
    # 每次測量前歸零，確保平均值正確
    total_distance = 0
    basic.show_icon(IconNames.TARGET)
    
    # 傳送開始訊號
    radio.send_string("--- START ---")
    basic.pause(200)  # 【優化】加長等待時間，確保通訊不塞車
    
    for index in range(5):
        # 讀取雷達數據並換算為公分
        dist = matrixLidarDistance.matrix_point_output(matrixLidarDistance.Addr.ADDR4, 3, 3) / 10
        
        # 組合訊息字串
        msg = "Count " + str(index + 1) + ": " + str(dist) + " cm"
        
        # 廣播每一次的測量結果
        radio.send_string(msg)
        basic.pause(200)  # 【優化】加長排隊時間，避免封包碰撞缺字
        
        # 累加距離
        total_distance += dist
        basic.pause(500)
        
    # --- 迴圈結束後，計算平均值並廣播 ---
    average = total_distance / 5
    avg_msg = "AVG: " + str(average) + " cm"
    
    # 傳送平均值
    radio.send_string(avg_msg)
    basic.pause(200)
    radio.send_string("-------------------")
    basic.pause(200)
    
    basic.show_number(average)
    basic.show_icon(IconNames.YES)

input.on_button_pressed(Button.A, on_button_pressed_a)

# --- 開機啟動設定 ---
avg_msg = ""
average = 0
total_distance = 0
# 雷達開機初始化 (讓雷達睜開眼睛)
matrixLidarDistance.initialize(matrixLidarDistance.Addr.ADDR4, matrixLidarDistance.Matrix.MAT)
# 設定廣播群組（對講機頻道為 99）
radio.set_group(99)
basic.show_icon(IconNames.YES)
