💡 研究動機與背景

南投縣（如集集、中寮、草屯等鄉鎮）是台灣特有種「石虎」最核心的棲地。然而，山區道路與棲地高度重疊，導致夜間「路殺」（Roadkill）事件頻繁發生。石虎屬於夜行性動物，其身上的斑紋在柏油路上具有極佳的保護色，在缺乏照明的山區道路，駕駛者往往受限於視覺死角，在發現目標時已避讓不及。這不僅造成珍貴物種的消逝，更嚴重威脅行車安全。

透過資料搜集，我了解到現行車輛的主動避障技術多依賴「超音波」或「影像辨識」。然而，超音波在高速行駛下的精準度受限，而影像辨識在極端全黑、強光或保護色干擾下容易產生誤判。為了解決這個痛點，我對「單點光學雷達（ToF 測距模組）」產生了濃厚興趣。光學雷達具備不受環境光干擾、反應速度快且測距精準的優勢。

本研究結合 MicroPython 程式撰寫，試圖在縮小比例的實驗環境中，模擬石虎在夜間出沒的情境。我參考了「送餐機器人」規劃路徑的原理，導入「網格座標路徑規劃」演算法，希望探討系統如何在「及時煞車」與「動態避障」之間找到最佳平衡點。

本研究不僅是科技的探究，更是一項具備在地情懷的實驗，期許能提出一套低成本且高效能的技術方案，為南投縣的生態保育與山區行車安全貢獻一份心力，讓石虎的生命不再因科技的侷限而在黑夜中凋零。

🌟 預期成果與貢獻

本研究預期能證實光學雷達在夜間完全黑暗環境下，測距精度優於傳統紅外線或超音波感測器。透過「路殺預警系統」的雛型開發，結合自動緊急煞車（AEB）與動態繞道邏輯，為未來智慧交通提供一個平衡「生物保育」與「行駛效率」的技術藍圖。

🚀 未來進一步研究方向

計畫導入 AI 影像辨識鏡頭（如 HuskyLens 哈士奇AI視覺感測器）。透過「雷達測距」與「影像辨識（石虎特徵）」的感測器融合（Sensor Fusion），讓系統不僅能避障，更能精準區分障礙物是否為保育類動物，實現更高層級的精準守護。

📝 我的實驗計畫與設計

🎯 研究目的

本研究旨在探討單點光學雷達（LiDAR）應用於夜間野生動物防路殺系統的可行性，並結合微控制器實作動態避障邏輯，具體研究目的如下：

  • 驗證環境光源對測距的影響： 測試光學雷達在極端光源（如全黑、手電筒強光）下的測距穩定性，藉此突破傳統視覺影像辨識在夜間無照明山區的應用限制。
  • 探討保護色與材質的偵測極限： 測試光學雷達對不同顏色與材質（模擬石虎保護色毛皮）的反射訊號干擾程度，釐理實體感測的極限與盲區。
  • 優化感測模組的最佳視角： 透過改變雷達的安裝高度，找出最適合精準偵測低矮、小型野生動物的硬體感測配置。
  • 建立安全預警與煞車門檻： 測量實驗車輛在不同行駛速度下完全煞停所需的物理距離，據此建立主動避障系統的安全啟動門檻數據。
  • 實作多情境動態避障演算法： 針對「受限車道（模擬山路）」與「開放空地（模擬廣場）」兩種情境，實作並比較系統在偵測到障礙物時，如何即時規劃出最短且安全的閃避軌跡。

🧰 研究設備與器材

  • 核心大腦： micro:bit V2.2 微控制器。
  • 實驗載具： 麥昆 Plus V3 智能擴充小車。
  • 測距核心： VL53L0X 單點光學雷達模組（ToF 感測器）。
  • 程式語言： MicroPython。
  • 模擬道具： 不同顏色的紙板、深色絨毛玩偶（模擬石虎）、捲尺、碼表、網格地墊、路徑標示膠帶。

⚖️ 實驗變因控制

為確保實驗具備高度的科學嚴謹性與結果的再現性，本研究於執行各項測試前，明確定義相關變因如下：

  • 控制變因： 相同的測試車道材質、同一台麥昆小車、同一顆 VL53LOX 光學雷達模組。
  • 操縱變因： 環境光源亮度、目標物材質與顏色、雷達安裝高度、小車行駛馬力（速度）、車道情境限制。
  • 應變變因： 雷達回傳之距離數值（cm）、測距誤差率（%）、車輛完全煞停距離（cm）、避障軌跡總長度與成功率。

🔍 實驗一：夜盲症測試（光線變因）

  • 步驟： 在筆直軌道上，分別設定 30、60、90、120 與 150 公分五種不同距離，並將模擬石虎的障礙物固定於對應位置。
  • 測試： 分別在「正常日光燈」、「手電筒強光」與「關燈全黑」三種環境下進行測量。每個距離皆反覆測試 5 次以計算「平均測量值」與「誤差率(%)」，藉此驗證光學雷達抗環境光干擾之能力與實際有效偵測範圍。

💡 秉澔的實驗後心得（已完成實測！）

「從這次實驗的數據裡，我發現了幾個很棒的結果：」

  • 🐾 石虎救星不怕黑（克服夜盲）：在「關燈全黑」的環境下，雷達依然能精準測量 30 到 150 公分的距離！這證明了 LiDAR 是依靠自己發射的光波來測距，完全不依賴環境光線。這代表在南投沒有路燈的漆黑山路中，系統依然能準確看見過馬路的石虎。
  • 🚗 無懼對向來車的「遠光燈」攻擊：這點最讓我驚豔！在「手電筒強光」（模擬對向來車的車頭燈直射）干擾下，測量數據依然非常穩定，150 公分時的誤差甚至只有 0.56 公分。這代表車頭燈的強光不會讓雷達「瞎掉」！
  • 📏 發現距離與誤差的物理極限：我觀察到，不論在哪種光源下，當距離拉長到 150 公分時，誤差值都會稍微變大（大約 3~4 公分）。這是因為光束變寬、反射回來的光變弱了，但在真實世界的煞車距離中，這點誤差是完全在安全範圍內的！
  • 🏆 最終結論：這份數據完美地證明了，我們選用的矩陣光學雷達，真的是非常適合用於夜間野生動物保育的完美感測器！

🔍 實驗二：保護色與材質陷阱（材質變因）

  • 步驟： 統一光源，並於筆直軌道上分別設定 30、60、90、120 與 150 公分五種距離。
  • 測試： 對比「白色紙板（淺色平滑）」與「黑色絨毛玩偶（模擬石虎深色長毛特性）」，每個距離皆反覆測試 5 次，觀察深色與絨毛材質是否會導致雷射訊號衰減、數值亂跳或產生極大誤差。

🔍 實驗三：最佳視角測試（高度變因）

  • 步驟： 將光學雷達安裝於小車上的三種不同高度：低位 (5 cm)、中位 (10 cm)、高位 (15 cm)，並設定 30 至 150 公分五種距離。
  • 測試： 每個高度與距離組合皆反覆測量 5 次，記錄成功測得精準距離的次數，藉此找出最能穩定偵測低矮型野生動物的最佳視角，避免光束由動物上方越過或掃瞄到地面產生雜訊盲區。

🔍 實驗四：生死一瞬間（速度與煞車距離變因）

  • 步驟： 設定小車以三種不同的行駛速度（20、40、60 cm/s）前進。
  • 測試： 觸發緊急煞車指令，反覆測量 5 次車輛因物理慣性至完全停止所需的「實際滑行距離」。此實驗所建立的安全煞車距離，將作為後續計算「極限避障觸發門檻」的關鍵數據。

🔍 實驗五：生命轉彎處（多情境動態路徑規劃）

  • 情境 A：受限車道（模擬山路）
    • 限制與測試： 小車在兩條平行白線內移動，設定不同的「預警觸發距離」。將石虎置於正前方、偏左側或偏右側，測試小車如何在不壓線的前提下，精準判斷並執行「向左或向右切換車道」的極限閃避。
  • 情境 B：開放空地（24 宮格地圖）
    • 限制與測試： 無邊界限制，於地面鋪設 4x6 (24宮格) 座標系統。測試小車在不同起點與石虎座標下，如何透過九宮格視野動態規劃出「行經格數最少」且「耗時最短」的避障與回歸路徑。
  • 比較分析： 以上情境各組合皆反覆實測 5 趟，記錄「避障成功率」、「行經總距離」與「耗時」，綜合評估兩種演算法之效能差異與應用場景。

⚠️ 研究限制

  • 硬體精度： VL53LOX 屬低價位教育版單點雷達，有效距離上限約 2 公尺，且無法進行 360 度環景掃描。
  • 模擬比例： 實驗採用縮小比例的麥昆小車，與真實重型車輛的慣性與煞車阻力不同，結果僅作為演算法邏輯參考。

2. 小車端（發射端）最終優化版程式碼

這是我們使用 MicroPython 撰寫的核心大腦！裡面包含了 LiDAR 讀取、毫米換算公分、計算平均值，以及為了解決通訊碰撞而加入的 200ms 無線電排隊廣播機制。

def on_button_pressed_a():
    global total_distance, average, avg_msg
    # 每次測量前歸零，確保平均值正確
    total_distance = 0
    basic.show_icon(IconNames.TARGET)
    
    # 傳送開始訊號
    radio.send_string("--- START ---")
    basic.pause(200)  # 【優化】加長等待時間，確保通訊不塞車
    
    for index in range(5):
        # 讀取雷達數據並換算為公分
        dist = matrixLidarDistance.matrix_point_output(matrixLidarDistance.Addr.ADDR4, 3, 3) / 10
        
        # 組合訊息字串
        msg = "Count " + str(index + 1) + ": " + str(dist) + " cm"
        
        # 廣播每一次的測量結果
        radio.send_string(msg)
        basic.pause(200)  # 【優化】加長排隊時間，避免封包碰撞缺字
        
        # 累加距離
        total_distance += dist
        basic.pause(500)
        
    # --- 迴圈結束後，計算平均值並廣播 ---
    average = total_distance / 5
    avg_msg = "AVG: " + str(average) + " cm"
    
    # 傳送平均值
    radio.send_string(avg_msg)
    basic.pause(200)
    radio.send_string("-------------------")
    basic.pause(200)
    
    basic.show_number(average)
    basic.show_icon(IconNames.YES)

input.on_button_pressed(Button.A, on_button_pressed_a)

# --- 開機啟動設定 ---
avg_msg = ""
average = 0
total_distance = 0
# 雷達開機初始化 (讓雷達睜開眼睛)
matrixLidarDistance.initialize(matrixLidarDistance.Addr.ADDR4, matrixLidarDistance.Matrix.MAT)
# 設定廣播群組（對講機頻道為 99）
radio.set_group(99)
basic.show_icon(IconNames.YES)
